## Top themes

1. **Admin audit-log coverage regressed on the two highest-sensitivity surfaces.** Compliance + Feature-flags pages don't audit-log their views. Every other Sprint-C admin page does. (F-Compass-02, F-Compass-03.)

2. **`admin_actor_user_id: null` across all admin destructive ops.** GDPR Art. 30 expects identity in the audit trail for admin-initiated deletions. The schema has the column; the actions never populate it. (F-Compass-01, F-Trace-07.)

3. **Three missing `<caption>` elements on data tables** regresses WCAG SC 1.3.1 — and these are exactly the surfaces (Compliance, settings/data) that auditors and regulators will read. (F-Halo-01, F-Halo-02.)

4. **Trust pages over-promise three controls that are not yet shipped:**
   - Mandatory admin TOTP + IP allowlist (admin-auth not wired). (F-Proof-03.)
   - Daily checkpoint signing of the audit chain (no checkpoint job exists). (F-Proof-04.)
   - DMCA designated agent registration (postal + phone TBD). (F-Proof-07.)
   These read as live in production today.

5. **Admin destructive-ops env-token gate is a plain `!==` compare** with no constant-time check, no rate-limit, no admin user_id. The token is the only thing between an authed admin session and "delete this tenant's data." (F-Trace-02.)

6. **GDPR cancel-deletion UI affordance is missing.** A user who soft-deleted gets the cancel link only in the grace-period email — the settings/data page doesn't show a cancel button on their `completed` deletion request. (F-Trace-03.)

7. **One hex literal sneaks in:** `apps/admin/app/(authed)/compliance/page.tsx:243` uses `#b51919` as a fallback for `--brand-red`. PRD §13.2 bans hex literals. (F-Canon-01.)
